# Interview Packet: Cybersecurity Analyst

**Role Family:** Cybersecurity Analyst  
**Universal Profile Fit:** Enterprise Technology Analyst — AI Automation, Cloud Security & Data Analytics  
**Typical Employers:** Toyota, JPMorgan Chase, Citi, AT&T, Capital One, Lockheed Martin

---

## Role Summary

Cybersecurity analysts monitor, detect, and respond to security threats — tuning SIEM alerts, reviewing access controls, supporting incident response, and maintaining compliance evidence. Enterprise roles require understanding of IAM, network security, and GRC frameworks.

## Technical Questions (Prepare Answers)

1. **"Walk me through an IAM policy review."**
   → Secure Cloud Evidence Lab: over-permissioned DevOps role → findings table → least-privilege recommendations.

2. **"How do you tune a SIEM alert to reduce false positives?"**
   → Lab example: adjust threshold, add time window, exclude service accounts, add geo enrichment.

3. **"What is least privilege and how do you enforce it?"**
   → Scope actions to specific resources; remove wildcard permissions; add MFA conditions; quarterly access reviews.

4. **"Describe the incident response process."**
   → Detect (SIEM) → Analyze → Contain → Eradicate → Recover → Lessons learned. Reference lab's IR playbook link.

5. **"NIST CSF vs SOC 2 — explain the relationship."**
   → NIST CSF = framework (Identify, Protect, Detect, Respond, Recover); SOC 2 = audit criteria (TSC). Crosswalk in lab Module 3.
